jQuery Events

jQuery is JavaScript so all the browser events are available with jQuery.

You can write your most of your event handlers with shorthand syntax such which looks like this $('.element.).click()...

...Or you can write your handlers with the on() method. Using on() gives you access to some additional event features and allows you to attach multiple events to an element with one statement.

The a modified version of the event object is available to jQuery event handlers. jQuery normalizes some event object properties to help with cross browser compatibility.

Preventing Default Behaviours

The browser runs various default behaviours on certain events. For example, when a user "clicks" a link the browser loads a new page, you fortunately do not have to write the JavaScript code to do that it is builtin to the browser.

Sometimes we want to "prevent" the browsers default behaviour. We can do this by using the builtinpreventDefault() method that comes with the event object.

Event Filtering

Attaching events to several elements on a page can be a performance hit.

To help with event performance, consider attaching an event to a parent element and then filter the event handler for a certain target or several target elements.

Modify the kitten gallery JavaScript code to attach a single event handler to the "thumbs" container and then filter for only clicks on the "a" elements.

Tip

To get the gallery to work set the CSS property of "pointer-events" to "none" on the thumb images so that the clicks on the thumb images are ignored and the clicks on the "a" are registered instead.
